## 📖 Обзор архитектуры

### DAP SDK - Distributed Application Platform

**Назначение:** Низкоуровневая платформа для создания распределенных приложений

**Ключевые компоненты:**
- **Core** - Базовые примитивы (логи, события, потоки)
- **Crypto** - Криптография (ECDSA, Dilithium, Falcon, Kyber512, SPHINCS+)
- **I/O** - Асинхронный I/O (epoll/kqueue/wepoll)
- **Stream** - Потоковая передача данных с каналами
- **Network** - Client/Server архитектура
- **Global DB** - Распределенная база данных (MDBX)

**Архитектура:** 8 уровней зависимостей, 0 циклических зависимостей

### Cellframe SDK - Blockchain Framework

**Назначение:** Высокоуровневый блокчейн фреймворк

**Ключевые компоненты:**
- **Datum** - Структуры данных (токены, указы, якоря)
- **Chain** - Базовый чейн с типами (DAG/Blocks/None)
- **Ledger** - UTXO ledger с балансами
- **Consensus** - Алгоритмы консенсуса (PoA, PoW, ESBOCS)
- **Governance** - Управление через указы (decrees) и политики
- **Network** - Сетевой слой с синхронизацией
- **Services** - Бизнес-логика (стейкинг, обмен, голосование, мосты)

**Архитектура:** 11 уровней зависимостей, 4 разрешенных цикла

## 🎯 Для разработчиков

### Добавление нового модуля

1. **Изучите [module_dependencies.md](module_dependencies.md)**
   - Определите уровень в иерархии
   - Проверьте существующие зависимости
   - Избегайте циклических зависимостей

2. **Следуйте принципам:**
   - Минимизируйте зависимости
   - Используйте Dependency Inversion для разрыва циклов
   - Применяйте forward declarations где возможно

3. **Обновите документацию:**
   - Добавьте модуль в матрицу зависимостей
   - Обновите диаграммы Mermaid
   - Документируйте новые зависимости

### Анализ зависимостей

```bash
# Генерация графа зависимостей CMake
cd build
cmake --graphviz=dependencies.dot ..
dot -Tpng dependencies.dot -o dependencies.png

# Анализ циклических зависимостей
cmake --trace-expand .. 2>&1 | grep "add_subdirectory"
```
