# Диаграмма зависимостей модулей DAP SDK и Cellframe SDK

**Дата обновления:** 2026-01-14  
**Версия:** DAP SDK 3.0-0, Cellframe SDK 4.0.0

## Обзор архитектуры

Система построена на двухуровневой архитектуре:
1. **DAP SDK** - низкоуровневая платформа для распределенных приложений
2. **Cellframe SDK** - высокоуровневый блокчейн фреймворк

## Матрица зависимостей модулей

### DAP SDK модули

| Модуль              | Зависимости                                          | Предоставляет                  |
|---------------------|------------------------------------------------------|--------------------------------|
| **dap_core**        | pthread, json-c, uthash                             | Базовые примитивы, события     |
| **dap_crypto**      | dap_core, XKCP, kyber512, secp256k1                 | Криптография                   |
| **dap_plugin**      | dap_core                                            | Система плагинов               |
| **dap_io**          | dap_core, dap_crypto, dap_net_common, pthread       | Асинхронный I/O                |
| **dap_net_common**  | dap_core                                            | Общие сетевые типы             |
| **dap_stream**      | dap_core, dap_crypto, dap_io                        | Потоковая передача             |
| **dap_stream_ch**   | dap_stream                                          | Каналы (gossip, pkt, proc)     |
| **dap_stream_event**| dap_stream                                          | События потоков                |
| **dap_stream_session**| dap_stream                                        | Сессии                         |

## Циклы зависимостей и их разрешение

### Идентифицированные циклы

1. **ledger ↔ mempool**
   - **Решение:** Используется forward declaration и разделение на интерфейсы

2. **chain_net ↔ ledger**
   - **Решение:** chain_net использует ledger через callbacks

3. **srv_stake → esbocs → srv_stake**
   - **Решение:** esbocs использует stake через интерфейс `dap_chain_esbocs_interface.h`

4. **governance (decree) ↔ policy**
   - **Решение:** policy зависит от decree (односторонняя зависимость)

### Стратегии разрешения циклов

1. **Forward Declarations** - объявление типов без определения
2. **Callback Interfaces** - использование функциональных указателей
3. **Dependency Inversion** - зависимость от абстракций, а не конкретных реализаций
4. **Event System** - асинхронное взаимодействие через события
